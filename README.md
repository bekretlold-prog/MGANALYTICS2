# 🍽️ Restaurant Analytics

Аналитическая система для ресторана. Работает прямо в браузере, данные хранятся в облаке (npoint.io).

## Что умеет

- 📊 **Дашборд** — выручка за 14 дней, каналы оплаты, топ блюд, алерты
- 📈 **Прогноз** — почасовой прогноз с учётом дня недели, тренда и прошлого года + потребность в персонале
- 🍽️ **Анализ меню** — ABC анализ, Menu Engineering матрица (звёзды/собаки), маржа по блюдам
- 📤 **Загрузка** — автоопределение типа отчёта, дедупликация

## Быстрый старт

### 1. Создай хранилища на npoint.io

1. Зайди на [npoint.io](https://npoint.io)
2. Создай **первый** документ с содержимым `[]` → скопируй ID из URL
3. Создай **второй** документ с содержимым `[]` → скопируй ID из URL

### 2. Настрой config.js

Открой файл `core/config.js` и заполни:

```js
NPOINT: {
  SALES: "ID_ПЕРВОГО_ДОКУМЕНТА",   // для продаж
  MENU:  "ID_ВТОРОГО_ДОКУМЕНТА",   // для меню
},
RESTAURANT_NAME: "Название твоего ресторана",
STAFF_NORM: 8,  // чеков на 1 сотрудника в час
```

## Какие отчёты загружать из iiko

| Отчёт | Название в iiko | Периодичность |
|-------|----------------|---------------|
| Почасовой | Отчёт по выручке с разбивкой по часам и типам оплаты | Ежедневно |
| Prod Mix | Отчёт о продажах блюд (с себестоимостью) | Ежедневно |

## Переезд на VPS (в будущем)

Когда будешь готов — меняешь только `core/db.js`.  
Вместо fetch к npoint — fetch к своему API. Всё остальное не трогаешь.
